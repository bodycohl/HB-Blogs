After reading Chapter 3 and 4 this weekend, I was still a little confused on why the box model was so prominent in CSS coding and style. I decided to use this as the focal topic of my blog post this week to give myself a better understanding of how to use it. The 'Box Model' refers to the idea that everything is CSS as a box around it, allowing you to create layouts and ensure alignment between elements. There are two types boxes: block boxes and inline boxes. Before reading this article, I was unsure what the differences were but I have learned that it impacts the way the box interacts with other boxes on the page as well as how the page layout is constructed. This is obviously an important component when it comes to creating an aesthically pleasing website since we can not have elements overlapping or with tons of extra space surronding them. One major difference is that block boxes will break to a new line and inline boxes will not. Both of these features can be useful for different circumstances. Another point that helped me understand this concept further was that each HTML tag has a default box type. Some examples are <h1> and <p> which are block box type and <a>, <span>, and <em> which are inline type. This idea that every element has a default type can help in understanding why each interact with other tags/elements like they do. Although each has a default type, there is a command in CSS to change the default type so that the element reacts in a different way. 

            element1 {
                display: inline;
                color: red;
                font-size: 10px;
            }
